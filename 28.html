UVA 10177 – (2/3/4)-D Sqr/Rects/Cubes/Boxes? 解题报告

Posted on 19 五月, 2011 by wanli — No Comments ↓

题目意思就是给出一个数字n，用这个数字拼成二维n*n的图形，三维n*n*n的图形，四维n*n*n*n的图形，分别计算这三个维数下能拼成的正方，长方等形的个数，相邻格子组合。。。。。（用int
WA了一次，改成long long AC了。。。。。）

规律：
正方个数(维数w)：result=1^w+2^w+3^w+…+n^w
长方个数(维数w): result1=(n*(n+1)/2)^w-result1
题目：

Problem J


(2/3/4)-D Sqr/Rects/Cubes/Boxes?

Input: standard input

Output: standard output

Time
Limit: 2
seconds

 

You can see a (4×4) grid below. Can you tell me how many squares
and rectangles are hidden there? You can assume that squares are
not rectangles. Perhaps one can count it by hand but can you count
it for a (100×100) grid or a (10000×10000) grid. Can you do it for
higher dimensions? That is can you count how many cubes or boxes of
different size are there in a (10x10x10) sized cube or how many
hyper-cubes or hyper-boxes of different size are there in a
four-dimensional (5x5x5x5) sized hypercube. Remember that your
program needs to be very efficient. You can assume that squares are
not rectangles, cubes are not boxes and hyper-cubes are not
hyper-boxes.

Input

The input contains one integer N
(0<=N<=100) in
each line, which is the length of one side of the grid or cube or
hypercube. As for the example above the value
of N is 4. There may be as
many as 100 lines of input.

 

Output

For each line of input, output six integers S2,
R2, S3, R3, S4, R4 in a single line
where S2 means no of
squares of different size in (
NxN) two-dimensional
grid, R2 means no of
rectangles of different size
in (NxN) two-dimensional
grid. S3, R3, S4, R4 means
similar cases in higher dimensions as described
before.  

 

Sample Input:

1
2
3

Sample Output:

1 0 1 0 1 0
5 4 9 18 17 64
14 22 36 180 98 1198


代码：
#include<iostream>

#include<fstream>

long long   two_diaz(long long
  n)
{
long long   re=0;
re=n*(n+1)*(2*n+1)/6;
return re;
}
long long   two_diac(long long
  n)
{
long long   re=0;
re=(1+n)*(1+n)*n*n/4-(n*(n+1)*(2*n+1)/6);
return re;
}
long long   three_diaz(long long
  n)
{
long long   re=0;
re=(1+n)*n/2;
re=re*re;
return re;
}
long long   three_diac(long long
  n)
{
return (long long )pow((1+n)*n/2,3)-three_diaz(n);
}
long long   four_diaz(long long
  n)
{
long long   re=0;
for(long long
  i=1;i<=n;i++)
{
re+=pow(i,4);
}
return re;
}
long long   four_diac(long long
  n,long long   a)
{
return (long long )(pow((1+n)*n/2,4))-a;
}
