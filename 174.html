包及访问控制权限


包的建立：

看如下代码：

package org.wl.demo01;

class Demo

{

public static void fun(){

System.out.println(“Hello Wolrld!!!”);

}

}

 

public class PackageDemo01{

public static void main(String args[]){

Demo.fun();

}

}

这里，第一行是定义一个包，那么编译的时候命令为:

javac -d . PackageDemo01.java

 


注意：这里有参数： -d . 不要忘了“.”，表示在当前所在的目录建包。


如果想在其他地方建包编译，只需要把“.”改成该目录即可例如想在D:JavaCode中建包那么：


javac -d D:JavaCode PackageDemo01.java

 

执行的时候不需要cd到目录，直接执行：

java org.wl.demo01.PackageDemo01

 

 


包的导入：

如果一个类想被外包所访问的时候必须得将其定义成public权限！

看如下代码：

package wl.demo01;

public class Demo01

{

public static void fun(){

System.out.println(“Hello World!!!”);

}

}

编译：javac -d . Demo01.java

再看下面代码：

import wl.demo01.Demo01;

 

public class Demotest

{

public static void main(String args[]){

Demo01.fun();

}

}

编译：javac Demotest.java

执行：

java Demotest

下面则正常输出Hello World!!!

如果想导入一个包中的所有类，那么直接使用“*”即可例如：

port wl.demo01.*;

如果不同包中的类名有冲突（类名一样），那么调用时直接使用类的完整名称：报名.类名。

 


静态导入：

当一个包中全部是静态方法的时候可以进行静态导入包，格式为：

import static 包;

那么在本类中使用静态导入包中的静态方法的时候不需要  类名.方法

而是直接 方法  即可。


jar命令：

在cmd中输入jar会出现如下;

E:JavaCode>jar

用法: jar {ctxui}[vfm0Me] [jar-file] [manifest-file] [entry-point] [-C dir] fil

s …

选项包括:

    -c  创建新的归档文件

    -t  列出归档目录

    -x  从档案中提取指定的 (或所有) 文件

    -u  更新现有的归档文件

    -v  在标准输出中生成详细输出

    -f  指定归档文件名

    -m  包含指定清单文件中的清单信息

    -e  为捆绑到可执行 jar 文件的独立应用程序

        指定应用程序入口点

    -0  仅存储; 不使用情况任何 ZIP 压缩

    -M  不创建条目的清单文件

    -i  为指定的 jar 文件生成索引信息

    -C  更改为指定的目录并包含其中的文件

如果有任何目录文件, 则对其进行递归处理。

清单文件名, 归档文件名和入口点名称的指定顺序

与 ‘m’, ‘f’ 和 ‘e’ 标记的指定顺序相同。

 

示例 1: 将两个类文件归档到一个名为 classes.jar 的归档文件中:

       jar cvf classes.jar Foo.class Bar.class

示例 2: 使用现有的清单文件 ‘mymanifest’ 并

           将 foo/ 目录中的所有文件归档到 ‘classes.jar’ 中:

       jar cvfm classes.jar mymanifest -C foo/。

 

那么最常使用的几个参数介绍：

‘C’ ：创建新的文档

‘V’ ：生成详细的输出信息

‘F’ ：指定存档的文件名

如果想将一个文件夹中的类打包重命名那么：

jar -cvf NEWNAME.jar FOLDER

例如：

E:JavaCode>jar -cvf test.jar wl

已添加清单

正在添加: wl/(输入 = 0) (输出 = 0)(存储了 0%)

正在添加: wl/demo01/(输入 = 0) (输出 = 0)(存储了 0%)

正在添加: wl/demo01/Demo01.class(输入 = 404) (输出 = 290)(压缩了 28%)

 

那么使用的时候必须得添加classpath。

例如：

set classpath=.;E:JavaCodetest.jar

 

那么使用和不打包的时候是一样的：
import wl.demo01.Demo01;

public class Testjar

{

public static void main(String args[]){

Demo01.fun();

}

}

 


访问控制权限：

private权限：只限本类访问。

默认权限：本包的类可以访问。

protected权限：在默认权限上加上本类的子类可以访问。

public权限：所有类都可访问。、
