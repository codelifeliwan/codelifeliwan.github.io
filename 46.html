frequently are used to generate sequences of pseudo-random numbers.

purposes to be really random. Random numbers are used in many
applications, including simulation.

As

this technique repeats after six numbers. It should be clear that
the longest sequence that can be generated using this technique is
limited by the modulus, M.

 

In
this problem you will be given sets of values
for Z, I, M,
and the seed, L. Each of these will have no
more than four digits. For each such set of values you are to
determine the length of the cycle of pseudo-random numbers that
will be generated. But be careful: the cycle might not begin with
the seed!

 

Input

Each
input line will contain four integer values, in order,
for Z, I, M,
and L. The last line will contain four
zeroes, and marks the end of the input
data. L will be less
than M.

 

Output

For
each input line, display the case number (they are sequentially
numbered, starting with 1) and the length of the sequence of
pseudo-random numbers before the sequence is repeated.

 

Sample
Input

 

7 5 12 4

5173 3849 3279 1511

9111 5309 6000 1234

1079 2136 9999 1237

0 0 0 0

 

Sample
Output

 

Case 1: 6

Case 2: 546

Case 3: 500

Case 4: 220
