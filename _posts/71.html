42-让Evolution有一点点像Outllook

微软把Outllok作为其Office套件一部分的做法，有些人可能对此有些异议。不管怎样，如果你已经使用了Outllok一段时间，开始
使用Evolution就会觉得有一点不适应。但是有一个好消息就是，你只需要进行一点点修改就可以把Evolution改造得和Outllok比较相像
了。

1、内联转发邮件。当你转发有“邮件收条”的邮件的时候，可能觉得有的麻烦（容易造成）。为了以文本格式转发，你可以点击“编辑”-》“首选项”-》“编写器选项”，从“转发方式”下拉框里面选择“引用”即可。

2、改变纯文本字体。如果你接受一个非HTML格式的邮件（譬如纯文本格式），Evolution默认使用Monospace字体来显示。你可以从
Evolution菜单“编辑”-》“首选项”-》“邮件首选项”里面，去掉勾选“使用与其他应用程序相同的字体”，把”等宽字体“里面选择”Serif
12″就可以了。

3、总是使用HTML格式化的Mail。当全世界都使用文本格式的邮件，人们就无法看到色彩缤纷的电子邮件了。如果想创建有色彩的邮件，必须使用HTML格式的邮件。点击“编辑”-》“首选项”-》“编写器首选项”，勾选“用HTML格式化信件”即可。

4、垂直的邮件预览窗口。现在的Outllok的界面都使用一个三栏竖直排列的窗口，邮箱窗口在左侧，中间是邮件，右侧是邮件预览。在Ubuntu，可以选择“查看”-》“预览”—》”竖直视图“就可以了。

 



43-配置静态IP地址

Ubunt的网络配置现在使用NetworkManager，它的工作非常好。但是如果使用无线网卡，并且使用DHCP方式分配IP的时候恐怕有点问题
（DHCP自动分配IP给其它工作站，这样可以简化IP管理方式。这样情况下，不需要配置IP，只要插上网线就可以进入网络工作了。但是如果使用Wifi
方式，一旦基站被密码加密，一切就完了）。你可能选择手工配置一个静态IP地址来解决这个问题。


在这样情况下，你可能希望使用Ubuntu原来的“网络配置”工具。它可以工作，但是在“漫游”模式下面有点问题。如果已经使用了秘籍41推荐的Wicd工具，就会发现它的功能是多么好用了。


使用“网络配置”工具，按照下面的步骤：


1、点击“网络配置”（在通知区域里面），选择“手工配置”，或者从“系统”-》“管理”——》“网络”选择，输入密码解锁。


2、双击”无线链接“。

3、在打开的对话框里面，不勾选”漫游模式“。如果是配置一个Wiki卡，输入你的基站用户名和密码，选择保护方式为”密码模式“。


4、选择IP地址方式为”静态IP地址“，输入IP、IP掩码和网关，点击OK。


5、在DNS标签里面，选择”添加“，输入DNS服务器的IP，可以添加多个。完成以后，重启以便设置生效。



44-格式化USB棒或者数码内存卡

 

有的时候，USB盘会停止工作，这个时侯最好的解决方法是重新格式化。在Ubuntu下面，可以安装下面的步骤.

 

1、插入U盘，一个图标会出现在桌面上，图标下面显示U盘的标签。

 

2、现在你可以查看U盘的信息了。打开终端，输入mount，会显示现在挂载的所有设备。

3、右键点击桌面的U盘图标，选择”卸载“。

 

4、回到终端，输入：

sudo cfdisk /dev/sdb（u盘的）

如果出现错误提示，输入：

sudo cfdisk -z /dev/sdb

 

5、现在输入”d“，删除U盘分区。然后输入”n“建立一个新分区，回车确认提示信息。输入”t“，再按回车，输入0C（注意是数字0然后加字母C），选择分区类型。输入”w

“,写入分区信息（如果提示已经有分区被标示为“可启动”，将被覆盖，不要管它）。

 

6、这是时候，如果U盘图标有出现的桌面上，依然右键卸载，然后在终端里面，输入下面的命令，可以建立一个FAT格式的分区（便于在Windows下面使用）：

sudo mkfs.vfat -F 32 -n USBSTIck -I /dev/sdb1

 

其中“USBSTICK”是U盘的标签，可以自己改变，/dev/sdb1就是前面的看到的U盘设备的Ubuntu设备编号。格式化完毕，一个新的U盘就会被自动加载到系统了。如果没有出现，再次拔出插入就可以了。



45-设置Ubuntu的Grub启动密码

在Grub启动的时候，你可以设定启动密码，锁定启动菜单，让系统无法启动。这个时候，可以按“P”键输入一个启动密码来解锁菜单，但是还是无法编辑启动项。

 

公平的说，这个方式无法防止用“Ubuntu安装CD”来启动系统，但是对于一些非技术型的用户还是能起到一定的包含作用。

 

打开终端，输入：

grub-md5-crypt

会提示输入一个密码。这个密码就是启动菜单解锁密码，所以请小心输入并牢记。可能需要输入两次，以确认输入正确。密码可以包含数字、字符和空格。

 

密码输入完成以后，会输出一个密码的加密字符串，现在记录下这个字符串。现在，可以把这个字符串启动选项里面。

 

打开/boot/grub/menu.lst文件：

gksu gedit /boot/grub/menu.lst

 

 

在文件的顶端，加入一行：

password –md5 $!$Qeb3b$X0.1bPvj47A#GEywBCR6m

 

其中“$!$Qeb3b$X0.1bPvj47A#GEywBCR6m”就是你刚才得到的字符串。

现在往下翻页，找到你的启动项，在启动项最后加入一行，内容是“lock”，譬如原来是：

 

title Ubuntu 8.04.1, kernel 2.6.24-19-generic

root (hd0,4)

kernel /boot/vmlinuz-2.6.24-19-generic root=UUID= …

initrd /boot/initrd.img-2.6.24-19-generic

quiet

 

现在变成了：

title Ubuntu 8.04.1, kernel 2.6.24-19-generic

root (hd0,4)

kernel /boot/vmlinuz-2.6.24-19-generic root=UUID= …

initrd /boot/initrd.img-2.6.24-19-generic

quiet

lock

 

现在把所有的启动项都加上lock行。

 全集之 6″ />

现在重新启动。启动以后，虽然可以用上下键选择启动选择，但是无法启动高亮的选项，并且提示消息：“Error 32：Must be
authenticated”。现在按“e”键也无法编辑。

 

想启动，必须按”p”键，输入密码（就是刚才设置的密码），现在就可以选择启动项来启动了。



46-转存虚拟终端文本到一个文件

 

如果你希望捕捉一个命令的输出，在终端窗口里面可以用复制/粘贴就可以了。如果在一个虚拟终端里面（譬如用putty连接的终端），如何做？

 

譬如下面的命令：

less a1.txt > output.txt

 

a1.txt文件不存在，所以会提示’a1.txt:没有该文件或者目录’，而output.txt不会被建立。

 

如果输入下面的命令就可以把错误内存转存到output.txt了：

less a1.txt > output.txt
2>&1

在output里面就是错误的内容了。

 

如果想捕捉全屏幕的信息，可以使用screendump命令(需要root权限）：

sudo screendump > output.txt



47-定制sudo/gksu的密码保持时间

 

默认情况下，Ubuntu会“记住”你的sudo/gksu密码一段时间，在这段时间以内，如果在使用sudo/gksu，就不会提示输入密码了，同时在这段时间以内，其他人也可以里面sudo的权限做任何事情了。

 

如果使用sudo的时候，使用“sudo
-K”模式，Ubuntu就不会自动记住sudo密码了，下一次输入sudo/gksu命令，就会还是要求输入密码。为了避免每一次都输入’-K’参数，可以编辑/etc/sudoers文件（一般不直接编辑此文件，而是使用
“sudo visudo”命令）。

 

找到”Defaults
env_reset”行（如果没有找到，就直接加入下面的一行），按’a’，加入“timestamp_timeout=0”，下面是修改后的显示：

Defaults env_reset,timestamp_timeout=0

 

按ESC，输入“:wq”，保存退出。

 

设置会立即生效。

 

如果，你想永久保存sudo/gksu密码，可以改为：

Defaults env_reset,timestamp_timeout=-1



48-在Windows下面访问Ubuntu文件

如果你是Ubuntu/Windows双系统，Ubuntu可以访问Windows下面的文件，但是Windows无法访问Ubuntu下面的文件，因为Windows没有支持ext2/3格式分区的驱动。

 

可能有一些驱动，可以让Windows访问Ext2分区。下面的软件是一个免费软件（非开源），你可以从http://www.fs-driver.org/download.html下载。在安装过程中，可能需要输入一个盘符（譬如Z：等）来挂载Ubuntu的Ext2分区。完成以后，就可以访问Ubuntu文件了。

 

如果希望使用开源软件，可以安装Explore2fs软件（在http://www.chrysocome.net/explore2fs)。Explre2fs需要在自己的程序窗口里面访问Ubuntu文件，同时在Vista下面无法工作（在XP下面正常）。

 

 

特别提示，一般不要在Windows里面改写Ubuntu文件，可能导致分区错误



（又出入，么得49）50-让Ubuntu对于儿童更安全

 

Ubuntu可以变成让儿童安全使用的操作系统，需要做两件事情。首先建立一个给儿童建立一个“受限用户”账户，防止他们无意间破坏系统。第二件事情就是，安装一个“Web内容过滤”软件，防止他们访问对孩子有害的内容。

1、建立一个受限用户

 

打开“系统”-》“管理”-》“用户和组”，点击“添加用户”按钮，输入用户名、真实姓名、密码，然后在“配置文件”下拉框里面选择“Unprivileged”。这样此用户就不可以建立新用户、不可以安装/卸载软件，但是可以正常登录，使用系统了。

 

2、安装Web内容过滤软件

 

Dansguardian是一个内容过滤软件，这意味这你将被过滤掉不良词语的Web请求，如果在google上搜索不良词语的网站也就没有结果显示。

 

安装Dansguardian，安装下面的步骤：

1、安装tiny proxy软件，这是Dansguardian需要的一个组件。然后安装Dansguardian。

 

2、两个软件安装完成以后，需要设置一下，编辑tinyproxy的配置文件：

sudo gedit /etc/tinyproxy/tinyproxy.conf

 

找到：

Port 8888

 

把8888改为3128.编辑完成，保持退出。

 

3、现在编辑Dansguardian的配置文件。

sudo gedit /etc/dansguardian/dansguardian.conf

 

找到含有“UNCONFIGURED”那一行。把“UNCONFIGURED”前面的“#”去掉，保存退出。

 

4、启动服务：

sudo invoke-rc.d tinyproxy restart

sudo invoke-rc.d dansguardian restart

 

你可能看到Dansguardian提示数据库过期，不要管它，以后它会自动更新。

 

5、配置上网代理

 

用刚才建立的小孩使用的账户登录，点击”系统“-》”首选项“-》”网络代理“。

 

6、选择”手工配置代理“，并且勾选”为所有的端口使用同样的代理“，在Http代理里面输入”localhost”，不要管端口，关闭退出。

 

 

现在依然用小孩账户登录，用Firefox打开一个成人网站（譬如http://www.playboy.com)，现在就会提示网络被限制了。

 

 



51-同时运行两个（或者更多）桌面

Ubuntu有一个“用户切换”功能，它的功能比起你第一次看到它想象的更强大。当切换到第二用户的时候，一个新的X
Server在已经启动的XServer基础上被启动。你可以用Ctrl+Alt+F7切换到第一个，用Ctrl+Alt+F9切换到另外一个。

 

如果需要，你可以手工开启更多的X Server。例如建立了一个账户，这个账户在一个虚拟终端里面登录了，输入：

$ startx — :1

一个新的GUI桌面就建立了。

 

现在可以建立更多的桌面：

譬如输入：

$ startx — :2

 

就可以用Ctrl+Alt+F10切换到此桌面了。



52 让所有的程序都可以全屏

 

有些程序有一个“全屏”模式，可以临时隐藏GNOME菜单和面板，获得更多的可视空间，并且去掉背景干扰。通常情况下，可以在“查看”菜单下面找到“全屏”菜单，但是有的程序并没有显示。

 

其实，可以定义一个全局快捷键，在“系统”-》“首选项”-》“键盘快捷键”里面，找到“切换全屏模式”行，定义自己的快捷键（建议Ctrl+Alt+f）。关闭窗口，现在就可以测试一下。这种方式几乎在所有的程序下面都工作，除了一些“对话框界面”的程序，譬如Ekiga.
