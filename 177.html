Java多线程


一、继承Thread类

Thread类是在Java运行时自动导入的java.lang包中的。无需自己手动导入。

看下面程序：

class MyThread extends Thread

{

String s;

public MyThread(String s1){

s=s1;

}

public void run(){

for(int i=0;i<10;i++){

System.out.println(s+”: i=”+i);

}

}

}

 

public class ThreadTest001

{

public static void main(String args[]){

MyThread A=new MyThread(“线程A”);

MyThread B=new MyThread(“线程B”);

A.start();

B.start();

}

}

 

这里，MyThread类是从Thread类继承来的，里面覆写了run方法，那么，如果直接调用run方法的话，肯定是先输出A的后输出B的，那么在程序里面使用的不是run方法，而是start方法，结果如下：

线程A: i=6

线程B: i=0

线程B: i=1

线程B: i=2

线程B: i=3

线程B: i=4

线程B: i=5

线程B: i=6

线程B: i=7

线程B: i=8

线程B: i=9

线程A: i=7

线程A: i=8

线程A: i=9

而且，每次运行结果不一样，也就是说，由系统默认的start方法来调用run方法，谁先抢到CPU资源就执行谁，每次谁先抢到CPU资源是不确定的【也就是并发执行】，所以导致了每次的执行结果不一定相同。


注：一个线程只能启动一次！！！


二、Runnable接口

看如下程序：

class MyThread implements Runnable

{

String s;

public MyThread(String s1){

s=s1;

}

public void run(){

for(int i=0;i<10;i++){

System.out.println(s+”: i=”+i);

}

}

}

 

public class ThreadTest001

{

public static void main(String args[]){

MyThread A=new MyThread(“线程A”);

MyThread B=new MyThread(“线程B”);

Thread t1=new Thread(A);

Thread t2=new Thread(B);

t1.start();

t2.start();

}

}

这里，MyThreard类继承了Runnable接口，Runnable里面有run方法，但是Runnable里面没有start方法，同时，Thread类里面有参数为Runnable的构造方法，所以由Thread来调用start方法。使用接口避免了单继承局限！


三、Thread类与Runnable接口

Thread类是Runnable接口的子类。

Thread类无法达到资源共享的目的，而Runnable接口则可以实现资源共享。

看如下程序：

class MyThread extends Thread

{

int x=5;

public void run(){

for(int i=0;i<10;i++){

if(x>=1)

System.out.println(x–);

}

}

}

 

public class ThreadTest001

{

public static void main(String args[]){

MyThread t1=new MyThread();

MyThread t2=new MyThread();

MyThread t3=new MyThread();

t1.start();

t2.start();

t3.start();

}

}

由于进程只能启动一次，那么在上面代码中想启动多个进程就必须得有几个对象，启动各自的进程，使用的是不同的对象的数据，运行结果如下：

5

4

3

2

1

5

4

3

2

1

5

4

3

2

1

 

而看下面代码：

class MyThread implements Runnable

{

int x=5;

public void run(){

for(int i=0;i<10;i++){

if(x>=1)

System.out.println(x–);

}

}

}

 

public class ThreadTest001

{

public static void main(String args[]){

MyThread t=new MyThread();

new Thread(t).start();

new Thread(t).start();

new Thread(t).start();

}

}

 

这个代码里面使用Runnable接口，虽然是使用的多个Thread对象，但是使用的却是一个MyThread对象的数据，因此实现了数据的共享，程序运行结果如下：

5

2

1

3

4

 


注：开发中一般使用Runnable接口！


四、线程的状态

1、创建状态：准备好了一个多线程对象。

2、就绪状态：调用了run方法，等待CPU分配资源。

3、运行状态：执行run方法。

4、阻塞状态：暂停进程执行，将CPU资源交给其他进程使用。

5、终止状态：程序执行完毕。
