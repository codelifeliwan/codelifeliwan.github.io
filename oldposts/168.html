单态设计模式：构造方法私有化


在Java中的构造方法私有化看下面代码：

class Test{

   
private Test(){}

}

 

public class Main{

   
public static void main(String args[]){

      
Test t=new Test();

   
}

}


这个代码编译结果如下：

Test()可以在Test中访问private

               
Test t=new Test();

为什么错误了呢？因为没有共有的匹配的构造方法供调用了，而在其他类里面又不能调用私有方法，所以就错了。

 


那么如果是在本类中调用是毫无疑问正确的，看下面代码：

class Test{

   
private Test(){

      
System.out.println(“私有构造方法调用！“);

   
}

   
Test t=new Test();

}

 

public class Main{

   
public static void main(String args[]){

      
Test t=null;

   
}

}

 


编译运行结果：

 

E:JavaCodegouzaofangfasiyouhuasrc>javac
Main.java

 

E:JavaCodegouzaofangfasiyouhuasrc>java
Main

 

E:JavaCodegouzaofangfasiyouhuasrc>

 


毫无疑问正确的。

 


那么为什么要有私有构造方法呢？看下面代码：

class Test{

   
private Test(){

      
System.out.println(“私有构造方法调用！“);

   
}

   
static Test t=new Test();

   
public void fun(){

      
System.out.println(“Hello World!”);

   
}

}

 

public class Main{

   
public static void main(String args[]){

      
Test t=null;

      
t.t.fun();

      
Test.t.fun();

   
}

}

 


运行结果：

E:JavaCodegouzaofangfasiyouhuasrc>java
Main

那么也就是我不需要真正实例化很多对象，只实例化一个对象，即是在本类中实例化的一个静态对象，即可以实现很多功能，当然也可以进行封装：

 

class Test{

   
private Test(){

      
System.out.println(“私有构造方法调用！“);

   
}

   
private static Test t=new Test();

   
public void fun(){

      
System.out.println(“Hello World!”);

   
}

   
static public Test function(){

      
return t;

   
}

}

 

public class Main{

   
public static void main(String args[]){

      
Test t=null;

      
t.function().fun();

   
   
Test.function().fun();

   
}

}

 


运行结果与上面一样。

 


其主要就是不需要实例化多个对象！！！即为单态模式！！！
