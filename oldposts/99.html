UVA 10047 – The Monocycle 解题报告

Posted on 28 七月, 2011 by wanli — No Comments ↓

题目意思比较绕，就是给出图，从起点’S’出发，到终点’T’，有这样的车轮：车轮每转90度，时间加1，上面的均匀5个扇形，从起点开始，是blue色扇形着地，方向向‘上’，然后求出从起点到终点的最后终点是blue扇形着地的最小时间。

本题是优先队列，广搜，等集合，在相同地方标记相同的颜色相同的方向不能重复访问，使用优先队列，每次找时间最小的进行扩展，配合广度优先搜索即可。
本题需要注意的地方很多，车轮可以经过终点再回去，也就是说车轮到达终点如果不满足条件可以继续走，知道满足条件为止，其次，变量较多，小心写错（我就是写错了一个WA了3次。。。。。）。还有最后要求相邻输出之间输出一个空行，不是在每个之后都输出空行。
还有就是为保证正确性，理论上是每次时间最多加1的扩展，但是这里算是UVA测试数据的一个bug吧，时间加2也过了。。。。。
题目：
Problem A: The
Monocycle 

A monocycle is a cycle that runs on one wheel and the one we
will be considering is a bit more special. It has a solid wheel
colored with five different colors as shown in the figure:

10047  -  The  Monocycle  解题报告” TITLE=”UVA  10047  -  The  Monocycle  解题报告” />

The colored segments make equal angles (72o)
at the center. A monocyclist rides this cycle on an 10047 - The Monocycle 解题报告” />
grid of square tiles. The tiles have such size that moving forward
from the center of one tile to that of the next one makes the wheel
rotate exactly 72o around its own center. The
effect is shown in the above figure. When the wheel is at the
center of square 1, the mid­point of the periphery of its blue
segment is in touch with the ground. But when the wheel moves
forward to the center of the next square (square 2) the mid­point
of its white segment touches the ground.

10047  -  The  Monocycle  解题报告” TITLE=”UVA  10047  -  The  Monocycle  解题报告” />

Some of the squares of the grid are blocked and hence the
cyclist cannot move to them. The cyclist starts from some square
and tries to move to a target square in minimum amount of time.
From any square either he moves forward to the next square or he
remains in the same square but turns 90o left or
right. Each of these actions requires exactly 1 second to execute.
He always starts his ride facing north and with the mid­point of
the green segment of his wheel touching the ground. In the target
square, too, the green segment must be touching the ground but he
does not care about the direction he will be facing.

Before he starts his ride, please help him find out whether the
destination is reachable and if so the minimum amount of time he
will require to reach it.

Input 

The input may contain multiple test cases.

The first line of each test case contains two integers M
and N (10047 - The Monocycle 解题报告” />,
10047 - The Monocycle 解题报告” />) giving the dimensions of the grid. Then follows the
description of the grid in M lines of N characters
each. The character `#‘ will indicate a blocked square,
all other squares are free. The starting location of the cyclist is
marked by `S‘ and the target is marked by `T‘.
The input terminates with two zeros for M and N.

Output 

For each test case in the input first print the test case number
on a separate line as shown in the sample output. If the target
location can be reached by the cyclist print the minimum amount of
time (in seconds) required to reach it exactly in the format shown
in the sample output, otherwise, print “destination not
reachable“.

Print a blank line between two successive test cases.

代码：
#include<iostream>
#include<queue>
#include<cstring>
#include<fstream>
#define maxn 30
using namespace std;
int m,n;
int beginx,beginy,endx,endy;
char map[maxn][maxn];
int direction[4][2]={{-1,0},{0,-1},{1,0},{0,1}};
int vis[5][4][maxn][maxn];//blue:0   white:1
  green:2   black:3
  red:4
typedef struct node{
int Time;
int dire;
int color;
int x,y;
friend bool operator <(struct node a,struct
node b){
return a.Time>b.Time;
}
}node;
int zheng(int a){
return a>0? a:-a;
}
int bfs(){
priority_queue<node>a;
node front,rear;
front.Time=0;
front.x=beginx;
front.y=beginy;
front.dire=0;
front.color=0;
a.push(front);
vis[0][0][front.x][front.y]=1;
while(!a.empty()){
rear=a.top();
a.pop();
int x=rear.x;
int y=rear.y;
for(int i=0;i<4;i++){
int nx=x+direction[i][0];
int ny=y+direction[i][1];
if(nx>=0&&ny>=0&&nx<m&&ny<n&&map[nx][ny]!=’#’){
node tmp;
if(rear.dire==i){
tmp.Time=rear.Time+1;
tmp.dire=i;
tmp.x=nx;
tmp.y=ny;
tmp.color=(rear.color+1)%5;
}else{
if(zheng(i-rear.dire)==2)
tmp.Time=rear.Time+2;
else
tmp.Time=rear.Time+1;
tmp.x=rear.x;
tmp.y=rear.y;
tmp.dire=i;
tmp.color=rear.color;
}
if(!vis[tmp.color][i][tmp.x][tmp.y]){
if(tmp.x==endx&&tmp.y==endy&&tmp.color==0)
return tmp.Time;
vis[tmp.color][i][tmp.x][tmp.y]=1;
a.push(tmp);
}
}
}
}
return -1;
}
int main()
{
int pcount=0;
while(cin>>m>>n){
if(0==m&&0==n)break;
if(pcount!=0)cout<<endl;
pcount++;
cout<<“Case
#”<<pcount<<endl;
memset(vis,0,sizeof(vis));
bool foundbegin=false;
bool foundend=false;
for(int i=0;i<m;i++){
cin>>map[i];
if(!foundbegin){
for(int j=0;j<n;j++){
if(map[i][j]==’S’){
foundbegin=true;
beginx=i;
beginy=j;
break;
}
}
}
if(!foundend){
for(int j=0;j<n;j++){
if(map[i][j]==’T’){
foundend=true;
endx=i;
endy=j;
break;
}
}
}
}
int re=bfs();
if(-1==re)cout<<“destination not
reachable”<<endl;
else cout<<“minimum time =
“<<re<<”
sec”<<endl;
}
return 0;
}
