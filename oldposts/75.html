113-安全的删除数据

 

手工安全的删除数据非常的有用，如果你打算卖掉你的电脑（甚至是丢弃它），你最好完全的销毁硬盘上面的数据。

 

简单的删除一个文件并不是非常安全，因为有很多软件可以恢复删除的文件，所以你必须用没有意义的数据填充整个磁盘，才能保证数据被永久删除了。

 

如果你不想永久删除一个硬盘里面的所有数据，也可以永久删除一个文件，保证这个文件的内容无法被复原。

 

Ubutu的“shred”就是一个解决方案。“shred”就是用随机数据覆盖文件或者磁盘的内容，保证这个文件的内容无法被复原。

 


永久删除磁盘

下面，我们将永久删除一个USB盘里面的内容，让它无法还原。有三个步骤：

1、找出这个USB盘在Ubuntu里面的位置。首先插入磁盘,看到桌面出现USB图标以后。然后开启一个终端，输入“mount”，查看所有已经挂载的磁盘。下面是我的电脑上面的显示，我的U盘的名称是KINGSTON,所有看到如下：

/dev/sdb1 on /media/KINGSTON type vfta (rw,nosuid,nodev, ..)

2、现在，右键点击图标，卸载USB盘。

3、现在，进入命令行：

sudo shred -v /dev/sdb

 

现在，shred开始删除时间。过程可能花费比较长的时间，但是也可以看到一个进度条。

 

默认情况下，shred覆盖数据25次，但是你可以通过参数“-n”来指定覆盖次数。但是除非是防止CIA来查看书籍，一个“-n1″参数就应该满足我们的需要了。

 

覆盖完成以后，你需要重新格式化此U盘。

 

如果你想覆盖一个硬盘的数据，你可以在Ubuntu安装盘的Live模式来进行这个工作。用Ubuntu安装盘启动电脑，桌面出现以后，打开终端，输入：

sudo swapoff

sudo shred -v /dev/sda

现在这个磁盘上面的所有操作系统（包括Windows分区）将被永久删除。如果需要删除软盘里面的内容，把”/dev/sda”改为”/dev/fd0″。


 


永久删除文件

相比删除磁盘，永久删除一个文件就比较简单了，不过需要添加一个“-u”参数。譬如想删除文件picture.jpg，可以使用：

shred -v -n1 -u picutre.jpg

现在不需要使用sudo命令。

117-防止“重复性劳损”

 

尽管有些人对于“重复性劳损”这个概念心存疑虑，但是它确实是长时间工作容易造成的伤害之一。如果想避免这个问题，可以安装WorkRave（一个GNOME
applet）-它会在一定的时间提醒你需要停止工作，放松一下了。安装这个软件，在“新立得”里面搜索并安装”workrave

“就可以了。安装完成以后，右键点击GNOME面板空白处，把WorkRave添加到面板即可。

 

WorkRave可以设置三个提醒时间：

“micro-breaks-设置在几分钟以内每几秒就提醒

“rest breaks”-设置每个小时提醒

“Daily limit”设置几天提醒一次

 

当到了提醒时间，WorkRave会弹出一个提示窗口，里面有一个计数器，帮助你计时。如果你实在太忙，可以点击“skip”按钮跳过。当提示时间完成，WorkRave的图标就会自动变成绿色。

 

提醒时间，在WorkRave的图标上面点击右键，选择“配置”就可以设置了

 



118-卸载Ubuntu

尽管可能有点不可思议，有些人发现对于Ubuntu已经不感兴趣，希望卸载Ubuntu，但是希望没有数据丢失。

 

在一个Windows/Ubuntu双系统的电脑上面，下面的步骤可以恢复Windows的启动项，然后删除Ubuntu分区，最后让Windows使用全部硬盘分区。

 

1、插入Windows安装CD，使用它启动电脑。当“选择安装”的菜单出现的时候，输入’r’进入修复终端，选择Windows分区，并且输入Windows的Administrator密码。

 

2、输入下面的命令，然后重新启动：

c:>fixmbr

c:>fixboot

c:>bootcfg /rebuild

c:>exit

 

3、现在使用Ubuntu LiveCD启动，在“System”-》“Administration”-》“Partition
Editor”找到Gparted。

 

4、找到Swap分区（名称是linux-swap),右键点击，选择“Swapoff”菜单。

 

5、找到Ubuntu分区，右键选择“删除”。然后删除Swap分区。

 

6、找到Windows分区，选择“Resize/Move”，把分区大小提示拖动到最大，选择Apply就可以了。或者建立一个新的Windows分区。

 

7、修正不正确的boot.ini选项。



119-无需动手，Ubuntu、Mac、Windows网络自由畅通

 

现在把三台电脑（分别是Ubuntu、Mac、Windows操作系统）放到一个地方，并把他们连接到一个Hub上面，现在无需配置，三台电脑就可以网络通讯了。这是因为，如果没有DHCP/DNS服务器提高IP，三个操作系统自动分配以169.254为开头的IP地址。

 



120-访问磁盘上面的ISO镜像

Ubuntu的标准分发模式是一个ISO镜像，你可以把它刻录进CD，然后用这个CD进行安装。看一个ISO镜像的内容有很多方法，你可以用”归档管理器“打开ISO文件，然后查看里面的文件列表，或者解压到硬盘查看具体内容。

 

另外一个方法是像一个真正的磁盘一样加载一个ISO文件。打开终端，譬如你的ISO文件的名称是ubuntu.iso：

sudo mkdir /media/ISO

sudo mount -o loop ~/ubuntu.iso /media/ISO

 

第一行是建立一个系统加载点（建立以后无需在建立），下一句是加载这个镜像文件。

如果想卸载镜像，可以使用：

umount /media/ISO

2007文件格式的争论，但是现在用的人也不多，就的.doc、.xls仍然广泛使用。OpenOffice.org可以打开Office

deb文件名根据自己的情况而定。请关注下载网页，更新这个软件，因为这个软件仍然在开放改进阶段。

125-Sleep，Ubuntu，sleep!

 

sleep命令通常使用在Shell脚本里面，但是如果想在提示里面进行延迟也非常有用。正如名字所示，他会让提示暂停一段时间。在一个命令的前面插入sleep，那么在这些这么命令以前会暂停计算机一段时间。譬如下面的命令，会让计算机在30s以后运行级别0（即关机）：

 

sudo sleep 30s; sudo telinit 0

 

注意，上面的命令设计sudo命令。sudo有一个权限“保持”时间（默认为2分钟），如果超过这个时间，这个命令不会只需成功。

128-让硬盘进入“省电”模式

Ubuntu有许多“电源管理”方面的特征，可以在“系统”-》“首选项”-》“电量管理”里面访问。但是如果你曾经用过Windows或者Mac，你可能注意到硬盘休眠现象。如果你一段时间以内没有任何操作，硬盘的电源就会关闭。当有任何数据请求，硬盘就会开始工作（可能有一点点延时）。

 

在Ubuntu下面，让硬盘进入“休眠”状态也是可以的，但是需要编辑一个配置文件。步骤如下：

1、配置文件的名称是hdpaarm.conf。可以用gedit比较这个文件：

gksu gedit /etc/hdparm.conf

 

2、找到“#spindown_time=24″这一行，把’#”去掉

3、spindown_time后面的数字可以是任意值。数值的单位是5秒，也就是说默认的开始休眠时间是24×5=120秒。但是如果数值超过240，那么单位改为30分钟，也就是说241代表30分钟，242代表60分钟，以此类推。把spindow_time=241作为配置项，可能比较合适，也就是30分钟没有操作硬盘，硬盘进入”休眠“状态。

 

4、现在保存，重启计算机使设置生效。

 

特别注意，这里的30分钟是30分钟没有硬盘操作，而不是你三十分钟没有使用计算机。如果存在后台进程，硬盘经不会进入”休眠“状态。

130-防止GNOME启动出错

 

我曾经不止一次的看到GNOME启动的时候，提示“There was an error starting the GNOME
Settings
Daemon.”这是时候，我总是通过删除/tmp这个文件夹来解决这个问题。现在注销，退出到登录屏幕。按下“Ctrl+Alt+F2”切换到另外一个终端，输入“telinit
2″关闭图形系统。

进入root，输入：

rm -rf /temp/{*,.*)

重启成功，就可以正常登录了。

 



131-在Ubuntu下面使用FTP

 

如果你在Ubuntu下面ftp登录一个站点，选择会非常的多。你可以使用Firefox、Nautilus文件管理器，还有命令行。

 


Firefox

使用Firefox，只需要在地址栏里面输入ftp站点的地址就可以了，但是必须以”ftp://”开头，而不是以”http://”开头。连接完成以后，就可以把文件拖动到桌面或者Natilus里面了，但是无法上传文件。

如果Ftp站点需要用户名和密码，Firefox会提示输入。你可以可以把用户名和密码以前输入到Url里面，格式是username:password@address，譬如：

ftp://keir:mypassword @ftp.eample.com

 


Nautilus

Natilus，可能是最免费而适用性更强的选择，它需要在本地和远程直接拖放文件来实现下载和上传。打开一个站点，打开Nautilus的文件浏览窗口，输入go-location。现在输入站点地址（注意包含ftp://)，输入用户名和密码，Nautilus在连接成功以后会自动记录这个地址。

 

连接完成以后，ftp站点就和本地文件系统一样，图标也会出现在桌面上。断开连接，右键点击桌面图标，选择”卸载卷“就可以了。如果你经常访问一个ftp站点，可以把这个站点加入到Nautilus的”书签“里面（使用菜单”书签“-》”添加书签“，或者使用Ctrl+d快捷键）。添加书签以后，Natilus在以后访问的时候会自动连接，如同ftp站点的内容在本地一样。

 


命令行工具

第三个Ubuntu提供的工具是命令行ftp工具。可以使用下面的命令连接：

ftp ftp example.com

输入用户名和密码进入。

现在进入ftp命令行操作其中：

cd-进入不同目录

get和put-下载和上传文件

”！“-挂起ftp，进入本地命令行，输入exit，重新进入ftp模式

help-显示可以使用的命令的帮助，help+command-显示具体命令的帮助

 

如果你使用过Windows里面的GUI
ftp程序-譬如CuteFTP等，你可以安装gftp。在”新立得“里面搜索并安装”gftp-gtk“,安装完成以后，可以在”程序“-》”互联网“里面找到它。

133-杀死”僵死“的进程

 

 

如果一个Linux用户希望杀死一个”僵死“的进程，可以立刻做到这一点。Kill命令的中途正在于此，但是它需要进程的ID作为参数才能工作。进程的ID可使用pgrep程序查找。譬如Firefox现在僵死，无法相应请求。打开一个终端，输入：

pgrep firefox

会返回数值，譬如是7198.现在输入：

kill 7198

就会杀死Firefox进程了。

你也可以使用killall命令。killall可以使用程序的名称，譬如输入：

killall firefox

 

kill还有一个图形界面的兄弟-xkill。打开一个终端查看，会出现一个提示光标，把光标在僵死窗户上面点击就可以了。如果改变了注意，可以点击右键就会取消xkill命令。xkill还可以结束GNOME的桌面组件，所有如果GNOME停滞响应，你可以用xkill结束。
