线程常用操作方法

几乎所有操作方法都在Thread类里面！！！

只要方法如下：

下面进行应用：

public class ThreadTest002

{

public static void main(String args[]){

Thread t=new Thread(“Test_Thread”);//实例化并设置线程名称

System.out.println(“The Thread name is : “+t.getName());//取得线程名称

t.setName(“TTT”);
//设置线程名称

System.out.println(“The Thread name is : “+t.getName());//取得线程名称

 

}

}

The Thread name is : TTT

线程强制运行：

把上面代码改改看下面代码：

 

 

class MyThread implements Runnable

{

public void run(){

for(int i=0;i<10;i++){

System.out.println(Thread.currentThread().getName()+” i= “+i);

}

}

}

 

public class ThreadTest002

{

public static void main(String args[]){

MyThread t=new MyThread();

Thread tt=new Thread(t,”Thread “);

System.out.println(“before Thread–>”+tt.isAlive());

try{

tt.start();

tt.join();

}catch(Exception e){

e.printStackTrace();

}

System.out.println(“in Thread–>”+tt.isAlive());

System.out.println(“after Thread–>”+tt.isAlive());

}

}

这里使用了join方法，其作用是强制运行某个线程，等待线程结束，结果如下：

before Thread–>false

Thread  i= 0

Thread  i= 1

Thread  i= 2

Thread  i= 3

Thread  i= 4

Thread  i= 5

Thread  i= 6

Thread  i= 7

Thread  i= 8

Thread  i= 9

in Thread–>false

after Thread–>false

 

可以看看in Thread这一行：

in Thread–>false

这个足以说明在运行System.out.println(“in Thread–>”+tt.isAlive());这行之前，线程已经运行完毕了！

 

线程休眠：

使用sleep方法，使线程休眠x毫秒【需要抛出异常】。

看如下程序：

 

class MyThread implements Runnable

{

public void run(){

for(int i=0;i<10;i++){

try{

Thread.sleep(1000);

}catch(Exception e){}

System.out.println(Thread.currentThread().getName()+” i= “+i);

}

}

}

 

public class ThreadTest002

{

public static void main(String args[]){

MyThread t=new MyThread();

Thread tt=new Thread(t,”Thread “);

System.out.println(“before Thread–>”+tt.isAlive());

try{

tt.start();

//
tt.join();

}catch(Exception e){

e.printStackTrace();

}

System.out.println(“in Thread–>”+tt.isAlive());

System.out.println(“after Thread–>”+tt.isAlive());

}

}

 

那么输出的时候是一秒一秒的输出的，每隔一秒输出一行。

 

线程中断，中断即不执行下面内容，用interrupt()方法。

 

后台线程：使用setDaemon()方法例如：t.setDaemon(true)就是设置t线程后台可以执行。
